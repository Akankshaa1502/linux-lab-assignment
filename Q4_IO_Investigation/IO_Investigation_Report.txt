=== Explanation of How Linux Manages File I/O ===
In Linux, everything is treated as a file, including regular files, devices, and pipes. Each process has a file descriptor table starting with 0 (stdin), 1 (stdout), and 2 (stderr) by default. The > operator redirects stdout to a file (overwriting it), 2> redirects only stderr, and 2>&1 merges stderr into wherever stdout currently points. The kernel enforces limits on how many file descriptors a process can have open simultaneously via ulimit, preventing a single process from exhausting system resources. Tools like lsof and /proc/pid/fd allow inspection of exactly what files and resources a running process is using, which is valuable for debugging logging and file access issues.
